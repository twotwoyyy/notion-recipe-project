> **버전**: 1.0.0  
> **작성일**: 2026-06-14  
> **상태**: 초안

---

## 1. 프로젝트 개요

### 1.1 프로젝트명

**레시피 모음** (Recipe Collection)

### 1.2 목적

Notion 데이터베이스를 CMS(콘텐츠 관리 시스템)로 활용하여, Notion에서 레시피를 작성하면 자동으로 웹 블로그에 반영되는 시스템을 구축한다. 별도의 어드민 패널 없이 Notion 인터페이스만으로 콘텐츠를 관리할 수 있어 운영 효율을 극대화한다.

### 1.3 핵심 가치

| 항목 | 내용 |
|------|------|
| 콘텐츠 작성 편의성 | Notion 에디터로 작성 → 웹 자동 반영 |
| 운영 단순성 | 별도 CMS 관리 페이지 불필요 |
| 확장성 | Notion 데이터베이스 구조 변경만으로 필드 추가 가능 |
| 성능 | ISR(증분 정적 재생성)로 빠른 페이지 로드 |

---

## 2. 목표 사용자

### 2.1 주요 타겟

- 레시피를 정리하고 공유하고 싶은 개인 블로거
- Notion으로 레시피를 이미 관리 중이며 웹 공개를 원하는 사용자

### 2.2 사용자 시나리오

```
[콘텐츠 작성자]
Notion에서 새 레시피 페이지 작성
  → Status를 "발행됨"으로 변경
  → 웹사이트에 자동 반영 (ISR 재검증)

[방문자]
메인 페이지에서 레시피 목록 확인
  → 카테고리 필터 또는 검색으로 원하는 레시피 탐색
  → 상세 페이지에서 재료, 조리법 확인
```

## 6. Notion 데이터베이스 구조

### 6.1 데이터베이스 속성

| 속성명 | Notion 타입 | 설명 | 필수 |
|--------|-------------|------|------|
| `Title` | `title` | 레시피 제목 | ✅ |
| `Category` | `select` | 카테고리 (한식, 양식, 중식 등) | ✅ |
| `Tags` | `multi_select` | 태그 (재료, 조리법 특성 등) | — |
| `Published` | `date` | 발행일 | ✅ |
| `Status` | `select` | `초안` / `발행됨` | ✅ |
| `Content` | page content | 레시피 본문 (Notion 블록) | ✅ |

### 6.2 Status 값 정의

| 값 | 동작 |
|----|------|
| `초안` | 웹에 노출되지 않음 |
| `발행됨` | 웹에 노출됨 |

### 6.3 카테고리 예시

`한식` / `양식` / `중식` / `일식` / `베이킹` / `음료` / `기타`

## 10. API 설계

### 10.1 Notion API 호출 패턴

```typescript
// lib/notion.ts 주요 함수

// 발행된 레시피 목록 조회
getPublishedRecipes(category?: string): Promise<Recipe[]>

// 단일 레시피 상세 조회
getRecipeById(id: string): Promise<Recipe>

// 레시피 본문 블록 조회
getRecipeBlocks(id: string): Promise<NotionBlock[]>

// 모든 레시피 ID 조회 (generateStaticParams용)
getAllRecipeIds(): Promise<string[]>
```

### 10.2 Notion 쿼리 필터 조건

```typescript
// 발행된 레시피만 필터링
filter: {
  property: "Status",
  select: { equals: "발행됨" }
}

// 카테고리 필터 (선택적)
filter: {
  and: [
    { property: "Status", select: { equals: "발행됨" } },
    { property: "Category", select: { equals: category } }
  ]
}
```

## 13. 제약사항 및 고려사항

| 항목 | 내용 |
|------|------|
| Notion API Rate Limit | 초당 3회 요청 제한 → 서버 캐싱 필수 |
| Notion 이미지 URL 만료 | Notion 호스팅 이미지는 1시간 후 만료 → `next/image` + ISR 재검증 주기 조정 필요 |
| 무료 Notion API | 통합(Integration) 권한 내 데이터베이스만 접근 가능 |
| Next.js 15 `params` | `await params` 필수 (Promise 반환) |
| lucide-react v1.17.0 | `Github` 아이콘 없음 → `ExternalLink` 대체 |
