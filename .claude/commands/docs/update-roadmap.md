# Claude 명령어: Update Roadmap

ROADMAP.md의 태스크 완료 상태를 현재 프로젝트 코드 상태와 비교하여 자동 업데이트합니다.

## 사용법

```
/docs:update-roadmap
```

## 프로세스

### 1단계: 현재 프로젝트 상태 수집

아래 정보를 **병렬로** 수집합니다:

- `docs/ROADMAP.md` 전체 읽기
- `docs/PRD.md`에서 기능 요구사항 ID (F-XX) 목록 추출
- 프로젝트 파일 트리 스캔 (app/, components/, lib/, types/)
- `git log --oneline -20`으로 최근 커밋 확인
- `git status --short`로 미커밋 변경사항 확인

### 2단계: 태스크별 완료 판정

ROADMAP.md의 각 태스크(`- [ ] **SX-X**`)에 대해 아래 기준으로 완료 여부를 판정합니다:

#### 파일 존재 확인
태스크에 명시된 파일 경로가 실제로 존재하는지 `Glob`으로 확인합니다.
```
예: "S3-1 RecipeCard 컴포넌트 — components/recipe/recipe-card.tsx"
→ Glob("components/recipe/recipe-card.tsx") 존재 여부
```

#### 함수/컴포넌트 존재 확인
태스크에 명시된 함수명이나 export가 해당 파일에 존재하는지 `Grep`으로 확인합니다.
```
예: "getPublishedRecipes(category?: string)"
→ Grep("getPublishedRecipes", path="lib/notion.ts")
```

#### 빌드 통과 확인
`npm run build` 또는 `npx tsc --noEmit` 결과를 확인합니다.
단, 이전에 이미 빌드를 실행한 결과가 대화에 있으면 재실행하지 않습니다.

#### 판정 기준
| 조건 | 판정 |
|------|------|
| 파일 존재 + 핵심 함수/컴포넌트 존재 + 빌드 통과 | ✅ 완료 |
| 파일 존재 + 핵심 함수/컴포넌트 존재 + 빌드 미확인 | 🔶 코드 완료 |
| 파일 존재하지만 내용이 불완전 | 🔶 진행 중 |
| 파일 미존재 | ⬜ 대기 |
| 외부 의존성 블로커 (API 권한 등) | ⏳ 블로커 |

### 3단계: ROADMAP.md 업데이트

판정 결과를 ROADMAP.md에 반영합니다:

1. **태스크 체크박스**: `- [ ]` → `- [x]` (완료 시)
2. **Phase 상태**: 마일스톤 요약 테이블의 상태 컬럼 업데이트
3. **Sprint 헤더**: 완료 시 `✅`, 진행 중이면 `🔶`, 대기면 유지
4. **완료 기준(DoD)**: 각 항목도 검증하여 체크
5. **Phase 마일스톤**: 하위 태스크 기반으로 업데이트
6. **전체 일정 요약**: 진행 바 업데이트 (`████`/`██░░`/`░░░░`)
7. **최종 업데이트 날짜**: 오늘 날짜로 갱신

### 4단계: 결과 보고

업데이트 완료 후 아래 형식으로 변경 요약을 출력합니다:

```
## 로드맵 업데이트 결과

### 변경 요약
| Phase | 이전 상태 | 현재 상태 | 변경된 태스크 |
|-------|----------|----------|-------------|

### 새로 완료된 태스크
- SX-X: {태스크명} — {판정 근거}

### 블로커
- SX-X: {태스크명} — {블로커 사유}

### 다음 작업 가능 태스크
- SX-X: {태스크명} (의존성 충족됨)
```

## 판정 규칙 상세

### Phase 완료 판정
- Phase 내 **모든 태스크**가 `[x]`이면 → `✅ 완료`
- Phase 내 **하나 이상의 태스크**가 `[x]`이고 나머지가 미완료 → `🔶 진행 중`
- Phase 내 **모든 태스크**가 `[ ]`이면 → `⬜ 대기`

### Sprint 완료 판정
- Sprint 내 모든 태스크 완료 → Sprint 헤더에 `✅` 추가
- Sprint 내 일부 태스크 완료 → Sprint 헤더에 `🔶` 추가

### 기술 메모 테이블
구현 중 발견한 새로운 기술 사항이 있으면 기술 메모 테이블에 추가합니다.

## 주의사항

- ROADMAP.md 외 다른 파일은 **수정하지 않습니다**
- 태스크의 설명 텍스트는 수정하지 않고 **체크박스와 상태 마커만** 변경합니다
- 판정이 애매한 경우 보수적으로 판정합니다 (완료보다는 진행 중으로)
- 외부 의존성 블로커는 별도로 표시하고 해결 방법을 안내합니다
